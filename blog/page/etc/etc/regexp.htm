## 정규표현식 (Regular Expression)  - validation(유효성검사) 등에서 사용 * 메타문자, 이스케이프 문자등을 이용하여 규칙의 문자열 표현 ## 문법 # meta기호 # 표현식 설명 ^ : 문자열 시작(carrat : 뒤에나오는 문자(pattern)가 시작에 있을때 $ : 문자열 종료 . : 임의의 문자 [단 \(역슬레시)‘'는 넣을 수 없습니다.] * : 앞 문자가 0개 이상의 개수가 존재할 수 있습니다. + : 앞 문자가 1개 이상의 개수가 존재할 수 있습니다. ? : 앞 문자가 없거나 하나 있을 수 있습니다. [] : 문자의 집합이나 범위를 표현합니다. -기호를 통해 범위를 나타낼 수 있습니다. ^가 존재하면 not을 나타냅니다. {} : 횟수 또는 범위를 나타냅니다. () : 괄호안의 문자를 하나의 문자로 인식합니다. | : 패턴을 OR 연산을 수행할 때 사용합니다. * escape 문자(자바에서는 \를 두번 써야함. >> \\) \s : 공백 문자 \S : 공백 문자가 아닌 나머지 문자 \w : 알파벳이나 문자 \W : 알파벳이나 숫자를 제외한 문자 \d : [0-9] 숫자 \D : 숫자를 제외한 모든 문자 (?i) : 문자열에서 대소문자를 구분하지 않습니다. (?=문자열) : 문자열을 포함함 (?!문자열) : 문자열을 포함하지 않음 * 어디에 사용될까요? 텍스트 처리 작업에 매우 유용합니다. 데이터 검증(주민번호, 휴대폰 번호등등)에 사용되기도 합니다. * UserName 소문자. 숫자. _-포함 3글자 이상 16글자 이하 ``` /^[a-z0-9_-]{3,16}$/ ``` * Password 소문자. 숫자. _-포함 6글자 이상 18글자 이하 ``` /^[a-z0-9_-]{6,18}$/ ``` * Email 소문자. 숫자. _-포함 @ 소문자와 . 2글자 이상 6글자이하 ``` /^([a-z0-9_\.-]+)@([\da-z\.-]+)\.([a-z\.]{2,6})$/ ```
