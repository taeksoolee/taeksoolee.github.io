## INDEX * 컬럼들을 기준으로 메모리 영역에 일종의 목차를 생성 * insert, update, delete (Command)의 성능을 희생하고 대신 select (Query)의 성능을 향상 # B_Tree 인덱스 구조 # 인덱스 선정 기중 * 카디널리티(Cardinality) : 해당 컬럼의 중복된 수치 >> 중복이 적을 수로 카디널리티가 높다고 함 # EXPLAIN 키워드 # mysql 인덱스 검색 // INDEX 목록 확인 (검색) > SELECT DISTINCT TABLE_NAME, INDEX_NAME FROM INFORMATION_SCHEMA.STATISTICS WHERE TABLE_NAME=[검색테이블] // 인덱스 검색 > SHOW INDEX FROM [테이블명 : 대소문자 구분함]
