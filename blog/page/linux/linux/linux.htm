## 리눅스 # 경로 /       : root ./      : 현재 ../     : 부모 ~/     : home $     : 마지막을 의미하는 경우가 많음 # 파일형태 파일형태    user    group            other -                      wrx             wrx             wrx -  : file d : directory l : link w : write ( 쓰기권한) r: read (읽기 권한) x : execute (실행 권한) ## 리스트 명령어 * 참고 : https://itholic.github.io/linux-basic-command/ ls -l(리스트 형태로 보여줌) -a(모두 보여줌) -i(인덱스를 보여줌) ## 디렉토리 / 파일 명령어 # 디렉토리 생성 $ mkdir [디렉토리 명] -p (--parents) 상위 경로 생성 -m (--mode=MODE) 권한 설정 (기본 755) -v (--verbose) 디렉토리 생성후 메시지 출력 # 파일 생성 $ touch [파일명] # 파일 / 디렉토리 삭제 $ rm -f [파일명] // * 완전 삭제(주의 할것) $ rm -rf [디렉토리 명] # 복사 $ cp -rp [복사할파일] [복사될 파일] -r 디렉토리 이하파일 -p 메타 데이터 함께 복사 // 대용량 파일복사 $ cp -rf [source] [target] $ rsync -anv --progress [sorce] [target] # 이동 $ mv [이동할파일] [이동될 파일] # 파일 또는 디렉토리 검색 명령어 $ sudo find / -name [파일명 디렉토리] # 실행파일 위치 검색 $ witch [실행파일명] ## 권한 관련 명령어 # 폴더 또는 파일 주인변경 $ chown # 폴더 또는 파일 권한변경 $ chmod ## 파일을 읽는 명령어 # cat [파일명] # tail [파일명] $ tail -F test.txt -F : 실시간 반영 # more # less : 대용량 파일 읽을 때 적합 ## link 파일 / 디렉토리 * --backup[=CONTROL] : 대상파일이 이미 존재할 경우에 백업파일을 만든 후에 링크파일 생성 * -b : 링크파일 생성시에 대상파일이 이미 존재하면 백업파일을 만든 후에 링크파일을 생성 * -d : 디렉토리에 대한 하드링크파일생성을 가능하게 함. 단 root 권한으로 수행하더라도 시스템의 권한제한으로 인하여 실패할 가능성이 높음. (-F 와 --directory 는 -d 와 동일함) * -f : 대상파일이 존재할 경우에 대상파일을 지우고 링크파일을 생성 * -i : 대상파일이 존재할 경우에 대상파일을 지울것인가를 확인요청 (--interactive 와 동일) * -s : 심볼릭 링크파일을 생성 * -S : 백업파일 생성시에 원하는 접미사(suffix)를 지정할 수 있음 * -t, --target-directory=DIRECTORY : 링크파일을 생성할 디렉토리를 지정 # 심볼릭링크 (Symbolic Link) * windows의 바로가기와 같은 개념 $ ln -s [원본파일] [링크파일] # 하드링크 (Hard Link) * 원본 파일과 동일한 내용의 다른 파일 $ ln [원본파일] [링크파일] ## 서버 시간(timezone) 변경 * 참고 : https://twpower.github.io/95-set-ubuntu-timezone # ubuntu // 국가 확인 (/usr/share/zoneinfo/Asia/Seoul) $ cd /usr/share/zoneinfo/ // 1번 방법 $ sudo ln -sf /usr/share/zoneinfo/Asia/Seoul /etc/localtime ## systemctl : 시스템 환경에 대한 기능을 제공 # 실행 확인 $ sudo systemctl status [프로세스명] ## 변수 설정( export ) $ export [변수명]=[값] $ export VARIABLE="value" $ echo $VARIABLE => vlaue ## network (네트워크) 명령어 # net-tools (네트워크 명령어 모음) 설치 # ping : 연결상태 확인 // 구글 DNS 서버로 ping 요청 = 서버 작동하는지 확인 $ ping 8.8.8.8 # route -n : 라우트 테이블 출력 (라우트) * Destination : IP * Gateway : gateway 주소 * Genmask : subnetmask * Flugs * Metric : 우선숭뉘 * Ref * user * Iface : 인터페이스 $ route -n // default(0.0.0.0) ip에 192.168.10.1 게이트웨이, enp0s3 인터페이스 추가 $ route add -net default gw 192.169.10.1 dev enp0s3 // default(0.0.0.0) ip에 192.168.10.1 게이트웨이, enp0s3 인터페이스 삭제 $ route del -net default gw 192.169.10.1 dev enp0s3 # ifconfig : ip 주소 정보를 출력 # nmap // 설치 $ sudo apt install nmap # netstat
